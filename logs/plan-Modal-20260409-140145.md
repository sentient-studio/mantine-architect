## 1. рџљ© Architectural Conflicts & Recommendations

### A вЂ” Component Cannibalization
**Severity: None detected**
The Figma shows a `Modal` shell that aligns directly with Mantine's `Modal` component вЂ” background, shadow, border-radius, close button, header, and overlay are all handled natively. We are building a thin wrapper that applies design-token overrides via `classNames` + CSS module; no custom overlay or dialog chrome is being hand-rolled.

The form fields visible in the Figma (TextInput, PasswordInput, Checkbox, Button) are rendered as story demo content only вЂ” not baked into the component. All use their Mantine counterparts correctly.

### B вЂ” Layout Paradox
**Severity: None detected**
The Figma's modal form body uses flex layout throughout (`display: flex; gap: 16px`). No hardcoded absolute coordinates. The two-column name row maps directly to `<Group grow>`, sections to `<Stack gap={rem(24px)}>`, and the horizontal footer to `<Group justify="space-between">`.

### C вЂ” Accessibility Tension
**Severity: рџ”µ NOTE**

**Issue 1 вЂ” "Have an account? Login" dimmed text contrast**
- Design Problem: `--text/dimmed` = `#868e96` (gray.6) on `#f8f9fa` (gray.0) background = **3.10:1** вЂ” fails WCAG 2 AA (requires в‰Ґ 4.5:1 at 14px).
- Framework Solution: Use `c="gray.7"` (#495057, 7.45:1 on gray.0) вњ…
- Trade-off: Slightly darker than the design token. Gray.7 is the established override for dimmed text throughout this project.

**Issue 2 вЂ” Required asterisk (*) on labels**
- Design Problem: `--text/error/default` = `#fa5252` (red.6) used for asterisks = ~3.5:1.
- Framework Solution: Accept Mantine's native `withAsterisk` behavior. The asterisk is a supplementary visual indicator вЂ” the required state is also communicated programmatically via `aria-required`. WCAG 1.4.3 excepts "incidental" decorative text. No override to Mantine's default `--mantine-color-error`.
- Trade-off: The asterisk remains at red.6 contrast, which is the standard Mantine pattern. Changing it would require overriding `--mantine-color-error` globally and would deviate from the Mantine design system.

### D вЂ” Thin Wrapper Docs Gap
**Severity: рџ”µ NOTE**
The component type is `Omit<MantineModalProps, 'classNames'>`. Storybook v10 cannot introspect external library types вЂ” the autodocs page will be blank without explicit `argTypes`.

Action: Plan includes explicit `argTypes` for all meaningful props (`title`, `size`, `centered`, `withCloseButton`, `withOverlay`, `shadow`, `padding`, `radius`) + a `Showcase` story using `args:` so the Controls panel works. Internal/required props (`opened`, `onClose`) hidden with `table: { disable: true }`.

## 10. Ambiguities Resolved

| Decision | Rationale |
|---|---|
| **Wrapper vs form component** | `Modal.tsx` is the wrapper shell only; form content lives in stories. A reusable "Modal" should accept any `children`, not be hard-wired to a registration form. |
| **`showTitle` / `showActions` Figma props** | Designer toggles for layout preview, not component props. `withCloseButton` + `title` (standard Mantine) cover `showTitle` behavior. `showActions` is story-level. |
| **Figma size в†’ Mantine size** | Figma xs=321px в‰€ Mantine xs=320px; Figma md=440px < Mantine md=500px. Slight delta acceptable вЂ” intent is responsive sizing via the Mantine scale, not hard-coded pixel values. |
| **Shadow** | Figma's 3-layer shadow doesn't match any Mantine preset exactly. `shadow="xl"` is the closest elevated-dialog shadow. Raw `rgba()` in CSS would violate CSS rules (no raw RGB). |
| **`--spacing/spacing-xl` = 24px** | Mantine's `xl` spacing = 32px. Use `gap={rem(24px)}` in the RegistrationForm Stack for exact fidelity. |
| **`classNames` removed from type** | Follows Drawer golden example pattern (`Omit<MantineDrawerProps, 'classNames'>`). Prevents consumer classNames from silently clobbering internal content/header/title CSS module overrides. |
| **`confirmPassword` validation** | Added to RegistrationForm (not in Drawer golden example but explicitly shown in Figma). Validates match against `password` field; adds test case 9. |
| **Footer layout** | Figma shows horizontal layout (link left, button right). Use `<Group justify="space-between">` вЂ” different from Drawer's vertical Stack to match the Figma grid more closely. |
| **`type="Default"` variant** | Generic placeholder with "Label" text labels вЂ” not a production variant. Implementing "Mantine" type only. |
| **Portal CSS variable scope (Gate 11)** | No custom CSS variables defined in `.root` that are expected to cascade into the portal content. `.content` uses only global Mantine color tokens. Gate automatically satisfied. |
