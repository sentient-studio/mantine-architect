## 12. Architectural Conflicts & Recommendations

### A — Component Cannibalization
**Severity: None detected**

The existing implementation correctly uses Mantine's `Table`, `Table.Thead`, `Table.Tbody`, `Table.Tfoot`, `Table.Caption`, `Table.ScrollContainer`, and `UnstyledButton` for sort controls. No bespoke table widget.

---

### B — Layout Paradox
**Severity: 🟡 ADAPT**

- **Design Problem:** Figma represents table structure as vertical Column Frames (one frame per column), each containing cell rows stacked vertically. This is Figma's display idiom.
- **Framework Solution:** HTML tables are row-based: `<tr>` contains `<th>`/`<td>` cells across columns. Data flows as an array of row records (`Record<string, React.ReactNode>[]`), not column arrays.
- **Trade-off:** Row-based semantics are required for accessible, correct HTML table rendering. Screen readers navigate by rows. `<th scope="col">` provides column headers. Mantine's Table handles all of this natively.
- **Delta from Figma:** Visual output is identical — the row-based DOM renders the same grid layout. No pixel delta.

**Additionally:** Figma shows two separate caption frames (one above, one below the content). HTML only supports one `<caption>` element per table. The single `caption` + `captionSide` prop is the correct web idiom.

---

### C — Accessibility Tension
**Severity: 🔵 NOTE**

- **Design Problem:** Caption text uses Figma variable resolving to gray.6 (#868e96), which has a 3.15:1 contrast ratio on white — fails WCAG 2 AA for normal text (requires ≥ 4.5:1).
- **Framework Solution:** Override via `classNames={{ caption: classes.caption }}` with CSS: `color: light-dark(var(--mantine-color-gray-7), var(--mantine-color-dark-2))`.
- **Override:** gray.6 (#868e96, 3.15:1) → gray.7 (#495057, 7.45:1) ✅
- **Already implemented** in existing component.

---

### D — Thin Wrapper Docs Gap
**Severity: 🔵 NOTE**

The component defines its own `DataTableProps extends BoxProps` with custom `columns`, `data`, and `footerData` props — Storybook CAN introspect custom interfaces. This is NOT a pure thin wrapper.

However, the `Showcase` story currently uses `render: () => <DataTable .../>` which prevents Controls from working. **Fix:** convert to `args:` format so the primary demo is interactive.

Additionally, `columns` and `data` are complex array types that are not useful as UI controls. Plan:
- `argTypes.columns = { table: { disable: true } }` — hide from controls panel
- `argTypes.data = { table: { disable: true } }` — hide from controls panel
- `argTypes.footerData = { table: { disable: true } }` — hide from controls panel
- Retain explicit `argTypes` for all boolean/enum props (`striped`, `highlightOnHover`, `withTableBorder`, `withColumnBorders`, `withRowBorders`, `captionSide`, `horizontalSpacing`, `verticalSpacing`)
